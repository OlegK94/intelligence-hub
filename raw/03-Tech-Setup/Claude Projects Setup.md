## Unterschied: Context-driven vs. Task-driven
- **Context-driven:** Café, Performance & Leben — akkumulieren Kontext über Zeit
- **Task-driven:** alle anderen — deliverable-orientiert, wenig State

## Schema je Projekt-Instruction
1. Zweck
2. Kontext (wo relevant)
3. Fokus
4. Rolle (mit Mindset)
5. Output-Standard
6. Nie (explizite Verbote)

## Global User Preferences (Highlights)
- Deutsch always, unabhängig von Input-Sprache
- Direkt, präzise, kein Consulting-Sprech
- Vollständige, sofort verwendbare Deliverables
- Keine neutralen Zusammenfassungen
- Keine Einleitungssätze die den Prompt wiederholen
- Annahmen immer explizit markieren
